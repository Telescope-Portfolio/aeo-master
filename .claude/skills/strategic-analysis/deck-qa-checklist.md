# Strategic Analysis Deck QA Checklist

Run this checklist before sharing any strategic analysis deck or document with stakeholders. Every unchecked item is a potential challenge from a skeptical board member or partner.

## Data Completeness
- [ ] Every chart has both count AND dollar versions (or explicit justification for showing only one)
- [ ] Every recommendation is backed by full-funnel data (acquisition + retention + expansion)
- [ ] Segment-level retention data (GDR, NDR, logo retention by tier) is included or flagged as a critical gap
- [ ] Missing data is called out explicitly as a limitation, not papered over

## Analytical Rigor
- [ ] Within-quarter monthly trends shown for the most recent quarter
- [ ] Every "this is working" claim includes the loss side (denominator): wins AND losses, count AND dollars
- [ ] Every channel/source is cross-tabbed by segment (e.g., Partner by SMB/MM/ENT, not just Partner aggregate)
- [ ] One-time events (migrations, partnerships ending, personnel changes) are labeled as such, not treated as structural trends
- [ ] Counter-arguments for each recommendation are stated explicitly
- [ ] Net revenue impact (won $ minus lost $) shown for every channel/source, not just win rate
- [ ] LTV math is shown for any segment where you recommend investment (acquisition cost vs. retention rate vs. ACV)

## Operational Accuracy
- [ ] Team/hiring recommendations verified against current state (check if role is already filled)
- [ ] Budget recommendations reference current spend levels
- [ ] Timeline claims are grounded in specific dates, not vague ("recently")
- [ ] Any "trend" claim specifies whether it's 3+ quarters of consistent direction or a single-quarter signal

## Presentation
- [ ] Every slide makes exactly one point
- [ ] The declarative title states the finding, not a category label ("Pipeline Volume Fell 52%" not "Pipeline Analysis")
- [ ] Source footer on every slide cites specific data source and sample size
- [ ] Every finding states which segment it impacts most
